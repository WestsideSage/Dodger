# Brief 4.2 — Season Preview

**Phase 8 Claude Design handoff. Logic settled in Phases 1–7.**

---

## 1. Teardown

**Component:** `frontend/src/components/match-week/command-center/SeasonPreview.tsx`
**Backing builder:** `src/dodgeball_sim/season_preview.py` → `build_season_preview`
**TypeScript type:** `SeasonPreview` in `frontend/src/types.ts`

**When it shows:** At the start of Week 1, before the Pre-Sim Dashboard, when `data.season_preview && !data.season_preview.skipped && !previewDismissed`. The player dismisses it to reach the command center.

**What exists today:**

The screen has three layout sections in a flex-column card:
1. An eyebrow ("SEASON PREVIEW") + the `top_goal` string as a headline. The goal reads e.g. "Finish in the top 4 of 7 to reach the playoffs."
2. Three stat tiles in a flex-wrap row: "Regular Season" (N weeks), "Your Bye" (Week N or "None scheduled"), "Playoff Cut" (Top N of N).
3. An optional strength/weakness pair in a flex-wrap row. Each is a left-border-accented card: "Roster strength" shows `strength.archetype` + `strength.avg_overall` avg OVR; "Watch area" shows the same for `weakness`. Both now use `archetype_display_name` (raw enum keys were fixed in Phase 1–7).
4. A skip-future-seasons checkbox + "To the Command Center" button.

**What is silent / redundant today:**

- The screen is informative (the synthesis confirmed the *content* is good) but visually indistinct — it reads as a generic stat card, not a season-opener moment.
- `top_goal` restates the playoff cut info that is already shown in the stat tile (redundant without offering a strategic interpretation). As primary copy it crowds the headline but doesn't add a motivating lens.
- The skip checkbox and action button share the bottom row but serve different audiences (one is a preference setting, one is the primary CTA). They compete visually.
- No visual calendar/timeline hint — the season length and bye week are rendered as isolated numbers, not as a shape the player can internalize (e.g., "you have 12 weeks before the cut, with a bye on Week 7").
- The strength/weakness cards are displayed even when both are null (the `&&` guard handles it), but when only one is present the flex-wrap leaves one slot visually empty.
- `avg_overall` is already an integer (Phase 6 float-leak fix) — confirm the rendering does not add a decimal.

**Phase 1–7 changes relevant here:**
- Archetype display names now render correctly (Phase 1–7 fix applied in `build_season_preview` via `archetype_display_name()`).
- `avg_overall` is returned as an integer from `weakest_position_group` / `strongest_position_group` (Phase 6 float-leak fix).
- The Season Preview is shown before the canonical fielded-6-powered readiness gates, so it has no dependency on those values.

## 2. Information Hierarchy

**Primary:** The season's shape — how many weeks, when the bye falls, what the playoff cut is. These three facts together form "the contract the player is signing up for." They should be immediately legible, not buried in a stat table.

**Secondary:** The strategic interpretation — the top goal sentence (reframe as an orientation line, not a headline duplicate), and the roster strength/weakness intel. These help the player form their Week 1 mindset.

**Cut:**
- The `skipped` boolean drives the skip checkbox behavior — it is a preference UI element, not content. It should be visually demoted (small label, not a prominent control).
- The `bye_week` raw integer is exposed both via `bye_text` and as the stat tile value — use `bye_text` only; never render the raw `bye_week` number directly.
- The `total_clubs` raw integer is embedded in `top_goal` prose — do not also display it as a standalone stat; it is context for the cut, not a standalone fact.

---

## 3. Data Inventory

**Payload fields available (from `SeasonPreview` in `types.ts`):**
- `regular_season_weeks`: integer — total regular-season weeks (e.g., 12)
- `bye_week`: integer or null — the week number when the player has no match
- `bye_text`: string — human-readable ("Week 7" or "None scheduled")
- `playoff_cut`: integer — number of clubs that make the postseason
- `total_clubs`: integer — total clubs in the league
- `top_goal`: string — server-rendered goal sentence ("Finish in the top 4 of 7 to reach the playoffs.")
- `strength`: `{ archetype: string, avg_overall: number }` or null — archetype is already display-name-resolved
- `weakness`: `{ archetype: string, avg_overall: number }` or null — archetype is already display-name-resolved
- `skipped`: boolean — whether the player has opted to skip previews in future seasons

**Derivable at render time:**
- Whether the player is in the top half or bottom half of the playoff cut: `playoff_cut / total_clubs`
- Whether there is a bye: `bye_week !== null`
- Whether strength and/or weakness data is available to show

**Fields intentionally absent (do not invent):** No per-opponent schedule, no win probability, no OVR for the whole team (only archetype-group averages). Do not add any of these.

---

## 4. Constraints

- Mobile viewport 390×844, no horizontal overflow. The current flex-wrap stat row already handles narrow widths — preserve this behavior.
- Semantic markup: wrap the three stat tiles in an appropriate landmark (`<section>`, `<dl>`, or `<ul>`) so screen readers can navigate. The current `div` + nested `div` structure is not machine-readable.
- No new dependency, no new API call. All data is on the `data.season_preview` object already loaded by the command-center endpoint.
- No routing change. This screen is a conditional overlay before the Pre-Sim Dashboard; it has no own URL.
- Skip preference: the `onSkipChange` callback posts to the server — the design must include the skip toggle; it just needs to be visually lower-weight than the CTA.
- The "To the Command Center" CTA must remain accessible without scrolling on a 390px-height-bounded viewport (current padding is generous; the redesign must not push the CTA off-screen at standard mobile height).

---

## 5. Success Criteria

1. A player arriving at Week 1 can answer three questions from the screen without reading: "How long is my season? When do I rest? How many teams make playoffs?"
2. The roster strength and weakness render even when only one is available (no orphaned half-row).
3. No raw enum key leaks in the archetype display (maintained from Phase 1–7 fix).
4. `avg_overall` displays as an integer, never a float (maintained from Phase 6 fix).
5. The skip checkbox and the CTA are visually distinguishable — user does not accidentally skip by clicking "continue."
6. No horizontal overflow at 390px width.
7. The screen reads as a season-opener moment, not a generic data card.
