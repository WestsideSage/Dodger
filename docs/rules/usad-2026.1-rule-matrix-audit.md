# USA Dodgeball 2026.1 Rule Matrix Audit

Source rulebook: `docs/sources/USA-Dodgeball-2026-Rulebook.pdf`
Gemini matrix: `docs/specs/usa_dodgeball_2026_extraction_matrix.md`
Audit date: 2026-05-20

## Result

Gemini's matrix has section-level coverage for all numbered rulebook sections 1-41. It is usable as the source extraction layer for V11, but it is not implementation-ready without corrections.

The main problem is not missing headings. The main problem is that several official clauses require deterministic domain models that the matrix sometimes collapses into physics, UI, or referee-AI language. V11 should preserve the matrix as a traceability aid and add a repo-specific rules architecture over it.

## Corrections And Gaps

1. Section 4 mixed-player interpretation: the matrix says `4M/2F is valid`. The rulebook says mixed teams may not play with four players of one gender. V11 should enforce no more than three starters of either gender in mixed games, while allowing 3-2 when only two players of one gender are available.

2. Section 7 bracket realism: the matrix correctly captures USAD bracket sizes and overtime, but the repo currently has a deterministic top-four playoff only. V11 should not promise full USAD bracket format unless the scheduler and standings model are also expanded.

3. Section 14 cloth throw clock: the matrix mentions cloth "play n balls" but under-specifies the penalty. The engine plan must record ball controllers at the call, attempts made inside the 5-second window, captain-selected additional outs, and official-selected fallback when no selection is made.

4. Sections 18, 20, 22 sequence finality: the matrix captures many examples, but V11 needs an explicit `SequenceOfPlay` resolver. A player hit by a ball is not always final immediately; later ball death, catches, out-of-bounds, and second-ball events can change eligibility.

5. Section 20 multiple-ball contact: the current engine cannot represent two active balls in sequence. The matrix includes examples, but the implementation plan must create a multi-sequence event ledger before attempting full multi-ball realism.

6. Section 22 catching: the matrix covers catch examples but should be stricter that only one valid catch per thrown ball can bring in one teammate, simultaneous catches can return multiple players, and same-sequence involvement can block re-entry.

7. Section 23 player entry: the matrix misses implementation emphasis on the 5-second entry window, back-line entry, live-on-contact status, and out-of-order queue penalties. These must be first-class queue events.

8. Section 24 ball retrieval: the matrix underplays clauses that are core to V11. Balls cannot be held in the queue, and entering players cannot touch/carry a ball before becoming live. These become deterministic forfeiture/out events. Designated retriever counts, sideline fights, nonparticipant interference, and spectator manipulation are advanced realism, not required for the first official-rules core.

9. Section 25 out-of-bounds: the matrix should distinguish a throw made while stepping out. It can relieve burden and can be caught, but it is not a live ball for hits.

10. Sections 33-41 discretion and discipline: the matrix sometimes proposes probability or "ref AI". V11 should not hide referee decisions behind opaque randomness. Use explicit `RuleDiscretionEvent` records with deterministic defaults and a visible reason.

## Rules That Need More Precise Engine Treatment

- Match and game are distinct. A match contains many games, different game scoring by material, late-start awarded games, and playoff overtime states.
- Ball ownership is distinct from ball liveness. A ball can be inactive, activated, live in flight, blocked, dead, retrieved, forfeited, contaminated, or replaced.
- Player status is more than alive/out. Official rules need active, hit-pending, exiting, queued, entering, live-entered, injured, carded, suspended, and ejected states.
- Burden is not the same as possession. Burden depends on ball majority, player majority for foam/no-sting ties, and referee reachability judgment for cloth ties.
- Throw clocks are nested inside burden and stoppage. A team can lose balls, lose players, or satisfy burden depending on valid throws and timing.
- Catching is a queue operation. It is not just a thrower-out outcome.
- No blocking is a mode that changes how held balls interact with the body. It is not a cosmetic replay label.
- Referee discretion must be logged as data. Ambiguous calls should have a deterministic default, a rule reference, and a replay-facing explanation.

## Audit Decision

Use the Gemini matrix as the extraction layer, but do not use it as the implementation plan. V11 should implement a smaller, deterministic official-rules core: material profile, shared official event envelope, rule-discretion events, match/game lifecycle, ball state, player state, sequence resolver, burden/clock, catch queue, Section 24 core retrieval clauses, no-blocking, and replay contracts. Discipline, injuries, interference, bracket expansion, and administrative rules should be staged behind the same domain seams.
