# Brief 4.5 — Rookie Class Preview

**Phase 8 Claude Design handoff. Logic settled in Phases 1–7.**

---

## 1. Teardown

**Component:** `frontend/src/components/ceremonies/RookieClassPreview.tsx`
**Backing type:** `RookieClassPreviewBeatPayload` in `frontend/src/types.ts`
**Beat key:** `'rookie_class_preview'`

**When it shows:** As a sequential beat in the offseason ceremony flow, directly before Signing Day (recruitment). It is the announcement of the incoming class before the player picks.

**What exists today:**

The screen has four visual sections:
1. A header block (eyebrow "INCOMING CLASS" + h2 "Rookie Class Preview").
2. A flex-wrap `dm-panel` with three large-number counters: `class_size` (Incoming Rookies), `top_prospects` (70+ floor, green when > 0), `free_agents` (Veteran Free Agents, gray).
3. An archetype breakdown panel: a flex-wrap row of `{name} ({count})` chips — one per archetype in the class. Hidden if `archetypes.length === 0`.
4. A market storylines panel: a bulleted `<ul>` of `storylines` strings. Hidden if `storylines.length === 0`.
5. An action bar: "Ceremony Control" + "Continue" button to advance to Signing Day.

**What is silent / redundant today:**

- The three large counters (class_size, top_prospects, free_agents) share equal visual weight. All three read as equally important, but `class_size` is the headline number and `top_prospects` is the most strategically relevant to the player. `free_agents` is the least interesting at this point (you know there are veterans; the count doesn't help you).
- `top_prospects` has a `title` attribute tooltip explaining what "70+ floor" means — this is valuable context buried in hover-only text. [Note: Mobile concerns are superseded, but relying solely on hover text is still bad for desktop accessibility].
- The archetype breakdown is a flat chip list with no signal about which archetypes are rare or abundant relative to last season. Without a baseline, "Thrower (4)" is meaningless — is that a lot or a little?
- Market storylines are plain bulleted text in the same visual weight as the archetype list. These are the most narrative elements on the screen but are rendered as utility text.
- No visual affordance that "this class includes players you have scouted." A player who spent weeks scouting specific prospects gets no recognition of that investment on this screen — the class preview and the player's own recruiting board are visually unrelated. (Note: the `RecruitmentChoice` beat that follows will surface their actual choices — this is structural, not a bug. But the brief should acknowledge it so the designer does not expect per-player scouting data here.)
- The "INCOMING CLASS" eyebrow is honest but flat for a ceremony beat. Offseason ceremony beats elsewhere have takeover-style headers (`CeremonyShell`); this screen uses inline styling only, not the `CeremonyShell` wrapper.

**Phase 1–7 changes relevant here:**
- Recruiting interest now transfers: the in-season board and the offseason signing pool target the same class year (Phase 1–7 fix). This makes the `RookieClassPreview` honest — the class the player is previewing is the same class they courted. Previously these could be disjoint.

---

## 2. Information Hierarchy

**Primary:** The class strength signal — `top_prospects` (70+ OVR floor count) is the single most actionable number. A player deciding whether to use their signing slots aggressively or conservatively needs to know if this is a strong or weak class.

**Secondary:** Class composition — `class_size` (total rookies entering the league) and the `archetypes` breakdown. Together these tell the player whether the class is abundant in their needed archetype.

**Tertiary:** Context and narrative — `free_agents` count and `storylines`. These are flavor; a player does not base a decision on them, but they make the screen feel like a real league with momentum.

**Cut:**
- The `free_agents` large number counter at primary visual weight — demote it to a secondary label or footnote. The player cares about rookies at this beat; veteran free agents are background information.
- The inline `title` tooltip on `top_prospects` — replace with visible explanatory text (e.g., "70+ OVR floor means scouts are highly confident") that renders on all devices.

---

## 3. Data Inventory

**Payload fields available (from `RookieClassPreviewBeatPayload`):**
- `class_size`: integer — total rookies entering the league this offseason
- `top_prospects`: integer — rookies whose scouted OVR floor is already 70+
- `free_agents`: integer — veteran free agents also available in the signing pool
- `archetypes`: `Array<{ name: string, count: number }>` — archetype distribution in the class; `name` values are already display-name-resolved (Phase 1–7 archetype fix applies to the generation path)
- `storylines`: `string[]` — server-authored narrative hooks about the class (may be empty)

**Beat base fields:**
- `beat.beat_index`, `beat.total_beats`: beat progress indicators
- `beat.title`: string — ceremony beat title
- `beat.body`: string — fallback prose (rarely rendered in the component; the component uses structured payload)

**Derivable at render time:**
- Whether `top_prospects > 0` (already used for green coloring — maintain)
- Whether `archetypes.length > 0` (already guarded — maintain)
- Whether `storylines.length > 0` (already guarded — maintain)
- `top_prospects / class_size` as a "class quality" percentage if a quality signal is needed

**Fields intentionally absent (do not invent):** No per-prospect names, no scouting data, no OVR distribution beyond the 70+ threshold count. This beat announces the class at a macro level; the `RecruitmentChoice` beat immediately following surfaces per-prospect detail.

---

## 4. Constraints

- Desktop-first matrix (1440x900 / 1366x768 / 1280x720). Mobile layouts are explicitly a non-goal. The flex-wrap counter row already handles narrow widths — preserve this behavior for desktop stress viewports.
- Semantic markup: the three counters should be marked up as definition list (`<dl>`/`<dt>`/`<dd>`) or as `<ul>` items with `role="listitem"` so screen readers can announce "Incoming Rookies: 18" etc.
- No new dependency, no new API call. All data is on `beat.payload`.
- No routing change. This is a sequential offseason beat.
- The "Continue" CTA must be reachable without scrolling on minimum desktop (1280x720); if storylines are long, ensure the action bar does not scroll off.
- The explanatory text for "70+ floor" must be visible without hovering (supersedes the mobile requirement, but still needed for clear desktop UX) — not a `title` attribute.

---

## 5. Success Criteria

1. The player can immediately read the class quality signal (`top_prospects`) as the most visually prominent number.
2. `free_agents` count is visible but not at the same visual prominence as `class_size` or `top_prospects`.
3. The tooltip explaining "70+ floor" is rendered as visible text on all devices (not hidden in a `title` attribute).
4. Archetype distribution is legible without the player having to count chips.
5. `storylines` reads as a narrative hook, not a utilities list.
6. No horizontal overflow at 1280x720 minimum desktop width.
7. The screen communicates "incoming class ceremony moment," consistent with the ceremony tone of adjacent beats.
