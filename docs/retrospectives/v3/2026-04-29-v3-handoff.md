# V3 Experience Rebuild Handoff

Status: Implemented and test-verified 2026-04-29. Manual screenshot review remains the main outstanding verification step before marking the milestone fully shipped in `docs/specs/MILESTONES.md`.

V3 focused on making the V2 manager loop feel more like a playable sports management game while preserving the integrity contract: event log first, visible inputs, logged RNG, uniform rules, and no renderer-driven outcomes.

## What Changed

### Pillar 1: Court Truth And Roster Integrity

- Added `LineupResolver.active_starters()` in `src/dodgeball_sim/lineup.py`.
- Made `build_match_team_snapshot()` cap full ordered lineups to `STARTERS_COUNT`.
- Updated `simulate_match()` to resolve full lineups, derive active starters, and store `home_active_player_ids` / `away_active_player_ids`.
- Updated stat extraction to use active IDs when present, keeping bench players out of match stat lines.
- Persisted match roster snapshots from Manager Mode with `match_role: active|bench` for replay/audit clarity.
- Updated report leverage calculations to use active participants instead of full rosters.
- Added roster UI separation between `Starters` and `Bench`.

### Pillar 2: Match Day And Replay

- Manager replay now has game-style controls: back, next, play/pause, speed buttons, key-event jump, and skip to final report.
- Replay event panel shows current actor/target/outcome plus probabilities and rolls when available.
- Replay MVP and final MVP labels now resolve display names instead of raw IDs.
- Match report adds a turning-point line and keeps primary performer rows name-based.
- Replay setup is derived from the active match setup, so the court cannot render recruited bench players as active bodies.

### Pillar 3: Pacing Controls

- Hub exposes pacing actions: `Play Next Match`, `Sim Week`, `Sim To Next User Match`, `Sim Multiple Weeks`, and `Sim To Milestone`.
- Bulk simulation uses the same deterministic match simulation and persistence path as individual match resolution.
- Added digest formatting with match count, week span, user-club results, standings context, notable performances, scouting/recruitment status, and next recommended action.
- Added `src/dodgeball_sim/sim_pacing.py` for pure stop-point selection and digest payload helpers.
- Current milestone support is strongest for playoffs and season end in the GUI. Pure helper coverage also documents recruitment/offseason/development/retirement-style stop reasons for future GUI expansion.

### Pillar 4: Hierarchy And Readability

- Hub action hierarchy now presents the pacing controls together around the next-match surface.
- Roster/lineup display explicitly separates starters from bench.
- Replay uses a larger court-first layout with a compact event detail panel.
- Bulk-sim output is a digest rather than a raw list of completed matches.

### Pillar 5: Writing, Names, And Flavor

- Generated rookie and prospect classes now enforce deterministic unique display names within a class, with fallback suffixes when random combinations are exhausted.
- Added `src/dodgeball_sim/copy_quality.py` with unresolved-token detection and title-label normalization.
- Friendly and manager report surfaces were tightened to prefer player names over raw IDs in primary presentation.
